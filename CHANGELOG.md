# Changelog

## Unreleased

- **Profiles: per-profile `model` rewrite.** New `model` field in `[profiles.<name>]` rewrites the JSON request body's `model` field on the way upstream. Lets the operator pin the right model name once (e.g. `model = "MiniMax-M2.7"` for the minimax profile) so callers never need `--model` or `ANTHROPIC_MODEL` for that profile. Capture events keep the ORIGINAL request body (with the caller's requested model) — corpus stays an honest record of intent. Skipped silently when body isn't JSON or has no top-level `model` field. Tests in `internal/proxy/profiles_test.go::TestProfileModelRewrite{,Skipped}`.
- **Profiles: fix base_url path joining.** When a profile's `base_url` had a path component (e.g. `https://api.minimax.io/anthropic`), the proxy was REPLACING the path with the request's restPath instead of joining. Result: requests to `/minimax/v1/messages` would forward to `api.minimax.io/v1/messages` (missing the `/anthropic` prefix). Fixed in handler.go to TrimRight base path of trailing slashes then prepend. Regression test `TestProfileBaseURLWithPath`.
- **Profiles — AWS-style multi-provider routing.** `marc --profile <name>` (and `MARC_PROFILE` env, or `default_profile` in config) now selects which LLM provider Claude is routed to. One install can talk to Anthropic, Minimax, OpenAI-compatible APIs, etc., all captured to the same shared corpus. Implementation is purely client-side — single proxy port (`:8082`), single `capture.jsonl`, no `marc-server` or ClickHouse changes. Routing is path-prefix: clauderun sets `ANTHROPIC_BASE_URL=http://<listen_addr>/<profile>`; the proxy parses the leading segment, looks up `cfg.Profiles[name]`, rewrites to the profile's `base_url`, and applies the matching `auth_style` (`x-api-key` passthrough or `bearer` rewrite). Per-profile `header_overrides` supported for providers that need pinned `anthropic-version` etc. Capture events do **not** include the profile — corpus stays provider-agnostic by design (no leakage into the active-learning loop). Resolution: `--profile` flag > `MARC_PROFILE` env > `default_profile` > `"anthropic"`. Configs without a `[profiles]` block auto-migrate from the legacy `[proxy]` + `[anthropic]` blocks on first load — existing installs keep working unchanged. New: `internal/clauderun/profile.go` (ParseProfileFlag, ResolveProfileName), `internal/proxy/profiles_test.go` (5-scenario routing test), per-profile reachability check in `marc doctor`. README §"Profiles (multi-provider routing)" added. Spec at `docs/profiles.md`.
- **Tightened question_gen prompt — anchor to source, no fabrication.** Initial telegram-delivered questions diverged from source events: the prompt's quality bar literally said "Name specific languages, file types, sizes, or constraints" which pushed the model to invent details (`5K entities in a game`, `ParseConfig with 6 unexported helpers`, `C++ particle simulator with 10 fields`) absent from the captured exchanges. New rules require situation/question/option_a/option_b to paraphrase the actual exchange, never add languages/frameworks/scales/characters not in the source, return `[]` when the event has no concrete decision (better silence than stretching). Domain-neutrality clause added so summaries about marketing/trading/ops aren't translated into engineering framing — same correction landed in `prompts/denoise.md`.
- **Active-learning feedback channel — Skip on Telegram is now a quality signal.** Added `sqlitedb.GetRecentByStatus(ctx, status, limit)` returning the N most-recent rows ordered by `COALESCE(sent_at, generated_at) DESC`. `marc-server generate` now appends two JSON arrays to its prompt: `skipped_examples` (recent rejections — anti-examples; "do NOT generate more like these") and `answered_examples` (questions the user bothered to answer — match this bar). Capped at 8 of each. `prompts/question_gen.md` was extended with a "User feedback on prior questions" section that explains the calibration. No new Telegram UI surface needed; the existing `[Skip]` button + `a <id> S` text command already produce the negative signal. Verified: after skipping Q2/Q3 (the fabricated-detail variants), the next generate cycle produced Q7/Q8 — direct paraphrases of the source events with no invented context.
- **Hardened client systemd units (post-T010 follow-up).** `marc-proxy.service` and `marc-ship.service` templates now render `User=`/`Group=` (resolved from `SUDO_USER` → `USER` → euid lookup so `sudo marc install` runs the daemon as the calling user, not root) plus `NoNewPrivileges=true`, `PrivateTmp=true`, `ProtectSystem=strict`, `ProtectHome=read-only`, and `ReadWritePaths=<homedir>/.marc`. Capture writes are now owned by the operator, not root. `templateData` gained `User`/`Group`/`HomeDir` fields and a new `resolveTargetUser()` helper.
- **`marc install --user`, `--target-dir`, `--skip-load` flags.** Linux installs can now target `~/.config/systemd/user/` without sudo (`marc install --user`). The original root-gate in `installSystemd` is now bypassed by either `--dry-run` or any custom `TargetDir` (printing templates / writing under `$HOME` shouldn't need root). Two new tests cover the `TargetDir` and `--dry-run` bypasses; `TestLinuxNonRootRejected` was updated to reflect the new contract (default target dir + non-root + non-dry-run still rejected).
- **`marc install` cobra cosmetics.** Added `SilenceUsage: true` so operational failures don't print cobra usage spam (matches the pattern already on `marc-server` subcommands).
- **GitHub Actions: self-hosted + artifacts.kanolab.io mirror.** Both `release.yml` and `ci.yml` switched from `runs-on: ubuntu-latest` to `runs-on: self-hosted` to use the existing kanoonthteam runner fleet. After GoReleaser publishes the GitHub Release, the seven release artifacts (4× marc binaries, 2× marc-server binaries, `checksums.txt`) are mirrored to artifacts.kanolab.io via `kanoonthteam/upload-artifact@v1` with 90-day retention. Workflows now use `concurrency` groups, `actions/setup-go@v5` with caching, pinned GoReleaser to `~> v2`, and idempotent `gcc-aarch64-linux-gnu` install. `release.yml` adds a `workflow_dispatch` input for re-releasing an existing tag.
- **ZFS snapshot ops bundle (T025).** Added `ops/marc-snapshots.cron` (the spec's weekly recursive snapshot, Sunday 03:00 local) and `ops/zfs-setup.md` covering: one-time pool/dataset creation with ashift=12, LZ4 compression, ARC tuning to 8 GB via `/etc/modprobe.d/zfs.conf`, mount-point repointing for `/var/lib/clickhouse` and `/var/lib/marc/state`, recovery scenarios for SQLite corruption / ClickHouse corruption / accidental table drop (rollback vs clone-and-extract patterns), and explicit non-recovery for disk failure (deferred to v2).
- **Default project seed in SQLite schema (post-T012 fix surfaced during e2e).** Added `INSERT OR IGNORE INTO projects (project_id, friendly_name) VALUES ('default','default')` to `applySchema` so freshly-generated questions inserted with `project_id='default'` no longer trip the FK constraint before an operator has configured per-project mappings.
- **Proxy SSE aggregation + gzip strip (post-T007 fix surfaced during e2e).** `internal/proxy/sse_aggregate.go` parses Anthropic SSE event streams (message_start / content_block_start|delta|stop / message_delta / message_stop) and synthesizes a non-streaming-shaped JSON response so downstream denoise sees `response.content[].text` instead of opaque raw bytes. The proxy now strips `Accept-Encoding` from forwarded requests so upstream returns plain SSE (gzip-encoded SSE was silently corrupting the captured response). Three new unit tests in `internal/proxy/sse_aggregate_test.go` cover the streamSSE-format input, the no-trailing-blank-line edge case, and a full text-block roundtrip including usage merge.

- **Architecture pivot — Telegram delivery moved from Go to Python (T020/T021).** `marc-server` no longer runs a Telegram bot; the existing openclaw `~/kanoonth/scripts/telegram-commands.py` proxy that already owns the chat handles all marc Telegram I/O. Reasoning: the proxy is on Cloudflare-tunnelled webhook mode and Telegram allows only one consumer per bot, so a parallel Go long-poller would conflict; reusing the proxy avoids a second BotFather token, a second webhook surface, and a second long-running process.

- Implemented `internal/generate` question-generation runner and wired `marc-server generate` (T018). `generate.Run(ctx, Options)` performs one cycle: (1) queries ClickHouse with the exact spec SQL (`SELECT event_id, project_id, summary, user_text, assistant_text, captured_at FROM marc.events WHERE has_decision = true AND captured_at > now() - INTERVAL 1 MONTH AND is_internal = false ORDER BY captured_at DESC LIMIT 30`); (2) assembles a prompt from the embedded `prompts/question_gen.md` plus the JSON-serialized event list; (3) invokes `claude -p --output-format json` as a subprocess with `ANTHROPIC_CUSTOM_HEADERS=X-Marc-Internal: true` in the environment (probe-verified on claude 2.1.119 — no HTTP fallback path needed); (4) parses the claude JSON envelope and the nested `[]CandidateQuestion` array; (5) filters by `DurabilityScore >= min_durability AND ObviousnessScore <= max_obviousness`; (6) inserts survivors into SQLite `pending_questions` with `status='ready'`; (7) updates `question_gen_cursor` to the latest `captured_at`. Non-zero claude exit → log + zero insert + nil (retry next hour). Empty CH result → no-op. `--max-turns` is dropped (does not exist in claude 2.1.119; `-p` is already single-turn). Added `sqlitedb.UpdateQuestionGenCursor(ctx, time.Time)` helper and `sqlitedb.DB.ExportDB()` for test access. 10 unit tests with manual fakes cover all 8 acceptance criteria.

- Implemented `marc-server init` subcommand (T013). `internal/initdb` applies ClickHouse DDL (`CREATE DATABASE IF NOT EXISTS`, `CREATE TABLE IF NOT EXISTS marc.events` matching spec ReplacingMergeTree schema with all 27 columns) and bootstraps SQLite via `sqlitedb.Open`. Idempotent — safe to re-run. `--check` flag queries `system.columns` and compares (name, type) pairs against expected; on drift exits 1 with column-level diff (`missing column: <name>` / `extra column: <name>` / `type mismatch: <name> got <a> want <b>`). The bootstrap connection targets `default` (not the user's database) so init survives a fully clean ClickHouse where the target database doesn't exist yet — clickhouse-go validates `Auth.Database` on first query, which would otherwise abort CREATE DATABASE. Wired into `cmd/marc-server init` with `SilenceUsage: true` so operational failures don't print cobra usage spam. - Implemented `marc-server process` daemon (T015). `internal/process` adds `Run(ctx, Options)` with the spec's 60-second poll loop.
